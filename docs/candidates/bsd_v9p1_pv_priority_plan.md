# BSD v9.1 Person/Vehicle Priority Plan

## Goal

Keep the current fast v9 deployment shape and recover BSD-critical
person/vehicle recall through data and conservative fine-tuning.

Frozen model constraints:

| Item | Value |
|---|---|
| Architecture | `YOLO26n_noattn` |
| Activation | `ReLU` |
| Input | `640x640` |
| Initial weights | `artifacts/bsd_v9_yolo26n_noattn_relu_640_from_v8_ft/weights/best.pt` |
| Board output | raw-logit `boxes/scores`, existing `yolo_decode.c` path |
| NB gate | `<5 MB` |
| Speed target | keep board NPU near v9, about `31-35 ms` |

The base-only v9.1 dataset is mainly a clean, reproducible starting point. Real
accuracy recovery should come from adding reviewed person/vehicle-heavy road
data through `--extra-set name:/images:/labels`.

Extra data priority:

| Priority | Data | Reason |
|---:|---|---|
| 1 | Road-scene person, especially small/occluded/side-rear/low-light | Recover person recall |
| 1 | Vehicle near/side/rear and hard negatives around guardrails/signs/poles | Stabilize vehicle recall and false positives |
| 2 | Rider/e-bike/motorcycle images where the person is visible | BSD risk still depends on detecting the person |
| 3 | vehicle-only images | Keep capped; do not let vehicle dominate |

## 2026-06-13 Result

Two v9.1 fine-tunes were evaluated after the first data cleanup:

| Candidate | Training validation | `bdd_proxy_val` P | `bdd_proxy_val` R | `bdd_proxy_val` mAP50 | PV Recall | PV mAP50 |
|---|---|---:|---:|---:|---:|---:|
| v9 ReLU from v8 | original route | `0.753` | `0.577` | `0.673` | `0.741` | `0.798` |
| v9.1 base | COCO early-stop, cleaned PV dataset | `0.720` | `0.602` | `0.677` | `0.760` | `0.797` |
| v9.1 bddval | BDD proxy early-stop | `0.744` | `0.593` | `0.680` | `0.743` | `0.798` |
| v7 fallback | stable reference | `0.682` | `0.615` | `0.674` | `0.786` | `0.808` |

Selected candidate for this stage: **v9.1 base**.

Reason:

- It improves BSD-critical person/vehicle recall over v9 from `0.741` to
  `0.760`.
- It keeps the deployable v9 structure: `YOLO26n_noattn + ReLU + 640`, raw-logit
  split ONNX, existing board decoder, and NB comfortably below `<5 MB`.
- The bddval early-stop variant has slightly higher overall mAP50, but its
  person/vehicle recall falls back to `0.743`, so it is not preferred for BSD.

Current ceiling:

- With the current dataset, the gap to v7 remains small but real:
  `PV Recall -0.026`, `PV mAP50 -0.011`.
- More early-stop or learning-rate variants are unlikely to create a large gain
  without new reviewed data.
- The next meaningful accuracy step is data addition: person-heavy road scenes,
  side/rear vehicle cases, and hard negatives around poles, signs, guardrails,
  shadows, and vehicle-like background structures.
