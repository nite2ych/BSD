报警判定 — 矩形 Zone × BBox IOU 比例

最简方案：区域=矩形，判定=交集面积/bbox面积 > 阈值

矩形 Zone 判定
判定代码（就这么多）
// 两个 AABB 交集面积 / bbox 面积
float bbox_zone_overlap(Box b, Box z) {
    float ix1 = max(b.x1, z.x1);
    float iy1 = max(b.y1, z.y1);
    float ix2 = min(b.x2, z.x2);
    float iy2 = min(b.y2, z.y2);
    float iw = ix2 - ix1;
    float ih = iy2 - iy1;
    if (iw <= 0 || ih <= 0) return 0;
    float bbox_area =
      (b.x2-b.x1) * (b.y2-b.y1);
    return (iw * ih) / bbox_area;
}

// 判定：overlap > 阈值即命中
if (overlap >= THRESHOLD)  // 0.3
    hit_zone(obj, zone_level);

更新后的 API

// 区域配置：矩形 + 级别
int alarm_set_zone(int    zone_level,     // 1/2/3
                    float  x1, y1,         // 左上角 (归一化 0-1)
                    float  x2, y2,         // 右下角 (归一化 0-1)
                    float  overlap_thr);   // 重叠阈值 (默认 0.3)

// 报警事件
typedef struct {
    int    zone_level;      // 命中最高级别区域
    int    class_id;        // 0=person, 1=bicycle, 2=motorcycle
    float  overlap;         // 实际重叠比例
    int    frame_count;     // 连续命中帧数
} AlarmEvent;

总结：3 个简化

  1. 4 点标定 → 存成矩形 (x1,y1,x2,y2)，标定阶段拖拽 4 点取外接矩形
  2. 射线法 → AABB 交集面积，一行代码
  3. 单点判定 → overlap 比例，阈值可配（默认 30%）
